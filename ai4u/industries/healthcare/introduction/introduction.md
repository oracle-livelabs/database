# Introduction

## Introduction
Healthcare prior authorization has become a bottleneck for diagnostic imaging and specialty medications. Clinical reviewers juggle fragmented EHR data, payer policies, and manual workflows that slow down care and frustrate patients. In this workshop you will build agents that automate the prior authorization journey end-to-end on Oracle Database 26ai.

You will step into the shoes of the HorizonCare coordination team. HorizonCare’s clinicians rely on fast approvals to keep patients on treatment plans, but their current process takes days. You will modernize this flow with AI agents that remember every interaction, surface the right evidence, and keep reviewers compliant.

### Objectives
* Understand the overall workshop structure and business problem
* Learn how agentic memory transforms prior authorization operations
* Preview the labs and key datasets you will use

### Prerequisites
* Basic familiarity with prior authorization terminology
* Comfort running SQL and Python notebooks provided in the labs
* Oracle Cloud account with access to Autonomous Database 26ai and OCI Generative AI

## Task 1: Meet HorizonCare
Review HorizonCare’s current prior authorization workflow, pain points, and success metrics. Note where delays occur and how they impact clinicians and patients.

## Task 2: Explore the Data Model
Scan the sample schema diagram that includes `PRIOR_AUTH_REQUESTS`, `PATIENT_PROFILE`, `MEDICAL_POLICIES`, and `AUTH_WORKFLOW_LOG`. Understand how each table supports the labs.

## Task 3: Plan Your Lab Journey
Map each lab to HorizonCare’s target outcomes: faster triage, transparent decisions, and persistent memory. Highlight the labs where you will build or extend these capabilities.

## Summary
You now understand HorizonCare’s business goals, the supporting data model, and how the labs align to those goals. Keep these objectives in mind as you build the system.
