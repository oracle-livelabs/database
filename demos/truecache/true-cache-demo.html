Welcome to Oracle True Cache

In-memory, consistent, and automatically refreshed SQL and object cache for Oracle Database 23ai

What is True Cache?

True Cache is an in-memory SQL and object cache Handles all read SQLs like Oracle Database and caches objects with automatic consistency deployed in front of an Oracle database. It's conceptually like a diskless Active Data Guard standby.

  • Always current and consistent data
  • Automatic schema propagation
  • No application changes required
  • Enterprise security built-in

Why True Cache?

Unlike conventional caches (Redis, Memcached), True Cache eliminates developer burden:

Cache Loading Automated
Data Consistency Automated
Complex Data Types Automated

Implementation Guide

Get started with Oracle True Cache in your environment

1. Configure True Cache Instance

-- Configure True Cache parameters
ALTER SYSTEM SET true_cache_memory_size = 10G;
ALTER SYSTEM SET true_cache_instance_mode = 'ENABLED';
ALTER SYSTEM SET true_cache_refresh_interval = 100;

-- Start True Cache instance
STARTUP MOUNT;
ALTER DATABASE OPEN READ ONLY;

2. Application Connection Configuration

-- Connection string for read/write splitting
jdbc:oracle:thin:@(DESCRIPTION=
  (CONNECT_TIMEOUT=90)(RETRY_COUNT=20)(RETRY_DELAY=3)
  (ADDRESS_LIST=
    -- True Cache for reads
    (ADDRESS=(PROTOCOL=TCP)(HOST=truecache-host)(PORT=1521))
    -- Primary database for writes
    (ADDRESS=(PROTOCOL=TCP)(HOST=primary-host)(PORT=1521))
  )
  (CONNECT_DATA=
    (SERVICE_NAME=myservice)
  )
)

3. Monitor Cache Performance

-- Check cache hit ratio
SELECT 
    cache_hits,
    cache_misses,
    ROUND(cache_hits / (cache_hits + cache_misses) * 100, 2) AS hit_ratio
FROM v$true_cache_stats;

-- Monitor cache memory usage
SELECT 
    current_size,
    max_size,
    ROUND(current_size / max_size * 100, 2) AS usage_pct
FROM v$true_cache_memory;

4. Java Application Example

import java.sql.*;
import oracle.jdbc.OracleConnection;

public class TrueCacheExample {
    public static void main(String[] args) {
        // Connection will automatically route reads to True Cache
        String url = "jdbc:oracle:thin:@truecache-scan:1521/myservice";
        
        try (Connection conn = DriverManager.getConnection(url, user, pass)) {
            // Read query - goes to True Cache
            PreparedStatement read = conn.prepareStatement(
                "SELECT * FROM products WHERE category = ?"
            );
            
            // Write query - goes to primary database
            PreparedStatement write = conn.prepareStatement(
                "UPDATE products SET price = ? WHERE id = ?"
            );
        }
    }
}
