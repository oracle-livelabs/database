# Configure SSO on Your Private Agent Factory

## Introduction

In this lab session, you will learn how to leverage the Single Sign On (SSO) and User Management features and use them to create users and assign specific access permissions.

The **Single Sign-On (SSO)** feature allows administrators to add SSO providers and seamlessly integrate them for accessing Oracle AI Database Private Agent Factory. The configuration form enables the entry of key information such as Provider, Client ID, API URLs, and connection details, making it easy to enable secure, streamlined authentication for all users.

The **User Management** feature provides a centralized dashboard for managing users in Oracle AI Database Private Agent Factory. Administrators can view key statistics at a glance, such as the total number of users, active users, administrators, and editors.

The User Directory lists all users who have accessed the system via Single Sign-On (SSO), along with their roles, statuses, and last login details. Actions are available for managing user roles and accounts directly from this page.

**Estimated time:** 10 minutes.

### Objectives

By the end of this lab, you will be able to:

- Configure SSO authentication in Oracle AI Database Private Agent Factory using OAuth credentials.
- Navigate the User Management dashboard to view, interpret, and manage user roles and access permissions within the application.

### Prerequisites

- An SSO provider and the necessary credentials (Client ID, Client Secret, Endpoint URLs, etc.) as required for OAuth integration.

## Task 1: Register SSO credentials on Oracle AI Database Private Agent Factory

Open Oracle AI Database Private Agent Factory and log in. In the sidebar, click the **SSO settings** section.

![Main dashboard of Oracle AI Database Private Agent Factory with the left navigation panel expanded. The SSO option under the Settings section is highlighted, and the Get Started page is displayed with pre-built agent options and a quick start guide on the right.](images/get_started_sso.png)

![SSO Settings screen in Oracle AI Database Private Agent Factory showing fields to configure OAuth Provider for Single Sign-On. The fields include provider, connection name, client id, client secret, SSO API base URL, access token URL, authorize URL, user URL, server metadata URL, log out URL, and a redirect URL. The 'Add Provider' button is displayed at the bottom right of the form.](images/sso_page.png)

Here, you can configure OAuth providers to enable Single Sign-On (SSO) for your organization. Complete the form with the credentials generated by your SSO provider. Required fields include:

- Client ID - `<your client ID>`
- Client Secret - `<your client secret>`
- SSO API Base URL - `<your Endpoint>`
- Access Token URL - `<your Endpoint>/oauth2/v1/token`
- Authorize URL - `<your Endpoint>/oauth2/v1/authorize`
- User URL - `<your Endpoint>/oauth2/v1/userinfo`
- Server Metadata URL - `<your Endpoint>/.well-known/openid-configuration`
- Log Out URL - `<your Endpoint>/oauth2/v1/userlogout`

Click **Add Provider**. A success message will be displayed. You can confirm the service status, disable it, or delete it on this screen.
